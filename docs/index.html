Kaeo LLM Proxy

View on GitHub Releases

Kaeo LLM Proxy is a C# Windows system-tray application that acts as an Ollama API-compatible proxy in front of one or more llama.cpp servers and other OpenAI-compatible upstreams. It lets Ollama clients talk to backends that expose a /v1/ API instead of the native Ollama API.

Why this exists

Many local AI tools expect the Ollama REST API, while llama.cpp and similar servers often expose an OpenAI-compatible API. This proxy sits in between, listening on localhost:11434 by default, rewriting requests and responses as needed, and routing each request to the correct upstream model.

Key features

  • Ollama-to-OpenAI/llama.cpp request and response translation
  • Transparent /v1/* passthrough for OpenAI-compatible clients and SDKs
  • Streaming and non-streaming chat/completion support
  • Streaming tool-call rewriting, including conversion of inline XML tool calls
  • reasoning_content compatibility fixes for clients that expect visible assistant text
  • Model name mapping with per-model upstream URL, timeout, API key, and behavior settings
  • Optional instruction set injection for model-specific system prompts
  • Streaming heartbeats and periodic upstream heartbeat monitoring
  • Automatic conversation summarization retries when context windows overflow
  • Portable Windows tray app with LiteDB-backed request history and Serilog app logging

Supported endpoints

Incoming endpointBehavior
GET /api/tagsReturns configured proxy model names
GET /api/psReturns enabled mappings as running models
GET /api/versionReturns proxy version metadata
POST /api/showQueries /v1/models/{model} with model-list fallback
POST /api/generateForwards to /v1/completions
POST /api/chatForwards to /v1/chat/completions
POST /api/embeddingsForwards to /v1/embeddings
POST /api/embedAlias of /v1/embeddings
/v1/*Transparent passthrough with model rewriting for JSON requests

  • Better streamed tool-call aggregation and XML tool-call conversion
  • Correct SSE framing so each data: payload is emitted as its own event
  • Streaming and non-streaming reasoning passthrough improvements
  • Expanded documentation and GitHub Pages support

Configuration highlights

  • ListenPort and ListenAddress configure where the proxy listens
  • The Logging section controls app logs, retention, and database location
  • Model mappings define the proxy-facing name, upstream model, upstream URL, timeout, API key, and feature toggles
  • Per-model settings can enable thinking compatibility, streaming heartbeats, summarization, instruction sets, and response redaction

Quick start

  1. Install the .NET 10 Desktop Runtime if needed.
  2. Run Kaeo LLM Proxy.exe; it starts minimised to the system tray.
  3. Open the dashboard from the tray icon.
  4. Add one or more model mappings that point to your llama.cpp or compatible upstream servers.
  5. Optionally configure instruction sets, heartbeats, and logging/redaction settings.
  6. Point your Ollama-compatible client at http://localhost:11434.

Networking and security

  • Default binding is localhost, which keeps access local to the machine.
  • Binding to 0.0.0.0 or a specific external IP may require administrator rights or a URL ACL reservation.
  • Prefer request/response redaction when prompts or outputs may contain sensitive information.
  • Use HTTPS upstream endpoints where possible if the upstream is remote.
