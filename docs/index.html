Kaeo LLM Proxy

View on GitHub Releases

A C# Windows system-tray application that acts as an Ollama API-compatible proxy, translating requests from any Ollama client to one or more llama.cpp servers running their built-in OpenAI-compatible /v1/ API.

Why this exists

Ollama clients (Open WebUI, Continue, etc.) expect the Ollama REST API. llama.cpp speaks a slightly different OpenAI-compatible API. This proxy sits in between — you point your clients at localhost:11434 and it routes each request to the right llama.cpp instance, doing all format translation transparently.

Features

  • Translates the Ollama API to llama.cpp's OpenAI-compatible format
  • Supports streaming (NDJSON) and non-streaming completions
  • Model name mapping — map any Ollama model name to the actual model loaded in llama.cpp
  • Per-mapping upstream URL and timeout — route different models to different servers
  • Request logging with LiteDB
  • Application logging via Serilog with rolling files
  • System tray application — no console window, always available in the background
  • Portable deployment — all data stored alongside the executable

Supported Ollama Endpoints

Incoming Ollama endpointForwarded llama.cpp endpoint
GET /api/tagsGET /v1/models
POST /api/showGET /v1/models/{model}
POST /api/generatePOST /v1/completions

System Requirements

  • Windows 10 version 22000 (21H2) or later
  • .NET 10 Desktop Runtime

Quick Start

  1. Download the latest release from GitHub Releases.
  2. Run Kaeo LLM Proxy.exe — it starts minimised to the system tray.
  3. Double-click the tray icon to open the dashboard.
  4. Add model mappings and point your Ollama-compatible client at http://localhost:11434.
